# v5.1.0 вЂ” Learning Paths + event discovery (2026-06-04)

Turns the event-registration site into a **guided learning journey**: admin-curated career
roadmaps, plus discoverability (format/level/topic) on events. v1 is intentionally lightweight вЂ”
**static published guides**, no per-member enrollment or progress tracking (the schema leaves room
to add those later without rework).

## Learning Paths (public)
- **`/paths`** вЂ” landing grid of published paths; **`/paths/{slug}`** вЂ” a phased, numbered roadmap.
- A path is an **ordered list of steps**; each step is either an **internal Yemdat event** (links to
  the event page) or an **external resource** (video/article/course/doc/other в†’ opens in a new tab).
  Steps can carry an optional **phase heading** ("Phase 1 вЂ” Foundations") and be marked optional.
- Fully bilingual (EN/AR, RTL) on the existing design system. New nav link **"Learning Paths /
  Щ…ШіШ§Ш±Ш§ШЄ Ш§Щ„ШЄШ№Щ„Щ‘Щ…"** (desktop + mobile).
- Flagship seeded path **"Start Your Data Career / Ш§ШЁШЇШЈ Щ…ШіШ§Ш±Щѓ ЩЃЩЉ Ш§Щ„ШЁЩЉШ§Щ†Ш§ШЄ"** (career-changer journey)
  вЂ” idempotent `LearningPathSeeder` (auto-attaches up to 2 real active events as live steps).

## Admin (Filament)
- **LearningPathResource** (Content в†’ Learning Paths, gated by the `posts` permission): bilingual
  details, level/topic, cover image, publish toggle, and a **reorderable Repeater** for steps
  (internal event picker vs. external link, conditional on `resource_type`).
- **EventResource** extended: `format` (event/workshop/course), `level`, `specialty` (topic) Selects
  + a collapsible **"Learning details"** section (`What you'll learn` + `Prerequisites`, bilingual).
  Events table gains a **format** badge column + filter.

## Event discovery & detail (public)
- **`/events` filter bar** вЂ” search + format/level/topic filters (server-side, GET params; invalid
  enum values ignored). Sections narrow together; clear-filters + "no match" empty state.
- **Event detail** вЂ” format/level/topic badges, **"What you'll learn"** (one bullet per line) and
  **"Prerequisites"** blocks, and a **"Part of these learning paths"** section linking back to each
  published path the event belongs to.

## Data model (additive migrations вЂ” safe on existing rows)
- `events`: + `format` (enum, default `event`), `level` (enum, nullable), `specialty` (slug),
  `outcomes_en/ar`, `prerequisites_en/ar`.
- New `learning_paths` (slug, bilingual title/summary/description, image, level, specialty,
  estimated_weeks, is_published, sort_order).
- New `path_steps` (learning_path_id, sort_order, phase_en/ar, **event_id** UUID FK в†’ events
  nullOnDelete, title_en/ar, url, resource_type, notes_en/ar, is_optional).

## Models
- New `App\Models\LearningPath` (locale accessors, `steps()`, `specialtyOption()`, `scopePublished`)
  and `App\Models\PathStep` (locale accessors, `isInternal()`, internal steps inherit the event's
  localized title). `Event` gains `format/level/specialty/outcomes/prerequisites` fillable +
  `outcomes`/`prerequisites` accessors, `specialtyOption()`, and `learningPaths()` (belongsToMany
  through `path_steps`, published only).

## Server deployment (Hostinger manual upload)
1. Upload changed PHP + Blade + lang files.
2. `php artisan migrate --force` (additive: adds event columns, creates `learning_paths` + `path_steps`).
3. `php artisan db:seed --class=LearningPathSeeder --force` (idempotent вЂ” seeds the flagship path once).
4. `npm run build` locally в†’ upload `public/build/*` (new Blade utility classes).
5. `php artisan optimize:clear` (re-`optimize` if you cache in prod вЂ” routes use a controller, so
   `route:cache` stays safe). Filament assets unchanged (no `filament:assets:publish` needed).

No new dependencies.
