# v4.9.1 — Flow polish (Phase 5b) (2026-06-01)

## What changed

Phase 5b tightens member-facing flows with consistent feedback, double-submit protection, and
a couple of accessibility fixes.

- **Global success/error toasts** — a new `<x-ui.toast>` (in the app layout) renders
  `session('success')` / `session('error')` / `session('status')` as a dismissible, auto-hiding
  (5s) toast with `aria-live="polite"` / `role="status"`. RTL-aware (anchors top-start). This
  replaces the bespoke full-screen "Success!" modal that lived only on the profile page, so all
  member flows now give the same feedback.
- **Loading state on every submit** — `<x-ui.button type="submit">` now shows a spinner and
  disables itself **once the form is actually submitting**, preventing double-submits and giving
  instant feedback. Because it's built into the shared component, every migrated form (login,
  register, contact, forgot-password, OTP/verify, profile edit, resubscribe…) gets it for free.
- **Accessibility** — the mobile nav toggle gained `aria-label`, `aria-controls`, and a reactive
  `aria-expanded`; the responsive menu got the matching `id`. Toast close button is labelled.

## Implementation note (subtle but important)
The loading state hooks the form's **`submit` event**, not the button's `click`. Disabling a
submit button *synchronously* inside a click handler cancels the native form submission (a
disabled control can't submit) — which would have silently broken every form. Hooking `submit`
disables the button only after submission is already underway. Added `[x-cloak]{display:none}` so
the spinner never flashes before Alpine initializes.
