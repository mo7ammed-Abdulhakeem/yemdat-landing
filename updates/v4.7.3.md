# v4.7.3 — Filament Admin: Permission Gating (2026-05-31)

## What changed

Gates the Filament resources by the app's granular admin permissions, so non-super-admins
see only the sections they're allowed to manage.

- New trait `App\Filament\Concerns\AuthorizesViaPermission` — a resource declares
  `permissionKey()`; `null` means super-admin-only. Wires `canViewAny/Create/View/Edit/Delete`
  to `User::hasPermission()` / `isSuperAdmin()`.
- Applied:
  - **Users** → super-admin only (matches the legacy `UserController`).
  - **Members** → `members`, **Contacts** → `messages`, **Posts** → `posts`
    (the keys the legacy controllers enforced).
  - **Broadcasts** → `broadcasts` (from v4.7.2).
  - Events / Trainer Requests / Membership Tiers / Email Templates remain open to any admin
    (the legacy controllers didn't gate these either).
- Super admins bypass all gates.

Tests cover it (limited admin → 403 on disallowed resources; super-admin → full access). 77 green.

## Files changed (deploy to production)
- `app/Filament/Concerns/AuthorizesViaPermission.php` (NEW)
- `app/Filament/Resources/{Users,Members,Contacts,Posts}/*Resource.php`

## Server deployment
Upload the files; `php artisan optimize:clear`. No deps, no migrations.
