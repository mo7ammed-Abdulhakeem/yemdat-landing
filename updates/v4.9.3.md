# v4.9.3 — Admin/data bug fixes (2026-06-02)

Three reported bugs, all rooted in the auto-generated Filament Member resource + a corrupted
membership-tier record.

## 1. Editing a member in admin wiped their password (→ "wrong credentials" on login)
The generated Member **form** had `TextInput::make('password')->password()->default(null)` with no
dehydration guard. Member casts `password => 'hashed'`, so **every Save re-hashed/overwrote the
password** — locking the member out. Fixed: password is now only written when the admin actually
types one (`->dehydrated(fn ($state) => filled($state))`, required on create, revealable, blank =
keep current). This was the root cause of "can't sign in as ds-test@example.com".

## 2. OTP fields exposed in admin
The generated Member form and table showed `otp_code` / `otp_expires_at` (internal auth artifacts).
Removed from both. Also tightened the form: `membership_type` and `gender` are now Selects, email
is unique-validated, raw `password` is no longer pre-filled.

## 3. Membership page 500'd when switched to Arabic
`MembershipTier::features_ar` for the **expert** tier held a JSON-encoded *string* (not an array),
so the array cast returned a string and `count($tier->features)` threw
`TypeError: count(): Argument #1 must be of type Countable|array`.
- `getFeaturesAttribute()` now **always returns a clean array**, tolerating legacy JSON-string /
  comma-joined data (splits + drops empties) — the page can never 500 on this again.
- The corrupted `expert.features_ar` record was repaired to a proper array.
- The Filament MembershipTier form's `features_en` / `features_ar` switched from `Textarea`
  (which saved a string and corrupted the array cast) to **`TagsInput`** (one tag per feature) so
  editing a tier can't re-corrupt it.

## Server note
The robust accessor means prod won't crash even if `features_ar` is still corrupted there. To make
the **expert** tier's Arabic features display cleanly, either re-save it via the admin (TagsInput)
or run an equivalent data fix. No migrations, no new dependencies, no asset rebuild required.
