# v5.7.0 — Installable PWA with offline content support (2026-06-20)

Yemdat is now an **installable Progressive Web App** with **automatic offline
support** — built for an audience on slow, intermittent, or expensive
connections. Visitors can **add the site to their home screen** (app icon,
splash, full-screen, no browser chrome), and any **public page they've already
opened keeps working with no signal**. Pages never visited fall back to a
**branded, bilingual EN/AR "You're offline" screen**. Frontend-only — no
migrations, no seeders, no new dependencies. A Vite rebuild ships with this
release and **must be uploaded to both public folders**.

## What's new
- **Install to home screen** — web app manifest with name, theme color (brand
  brown `#593E2D`), beige splash background, and a full icon set (192 / 512 /
  maskable for Android + Apple touch icon for iOS).
- **Automatic offline** — a hand-rolled service worker caches the app shell and
  every public page the visitor opens, so repeat visits load instantly and use
  almost no data; previously-viewed events, news, and learning paths stay
  readable offline.
- **Branded bilingual offline page** — for never-visited URLs while offline.
  Reads the saved locale (`localStorage['yemdat_locale']`, mirrored from the
  server locale on every page load) to render the correct language + RTL, and
  auto-reloads when the connection returns.

## How it behaves (caching strategy)
- **HTML navigations → NetworkFirst:** fresh content when online; the last-seen
  cached page when offline; the offline screen if nothing is cached.
- **`/build/*` hashed assets & images (`/storage`, `/icons`) → CacheFirst**
  (images bounded by a ~60-entry LRU cap).
- **Bunny CDN fonts → stale-while-revalidate.**
- **Personalized / one-time routes → NetworkOnly (never cached):** `/admin`,
  `/livewire`, `/my-profile`, auth routes, `/lang/*`, `/track`, `/unsubscribe`,
  certificate verification — so no stale or wrong-user content is ever shown.
  All non-GET requests (form submissions) always go to the network.
- **Updates:** the service worker is keyed by a `CACHE_VERSION` constant
  (`yemdat-v5.7.0`); on activation it purges caches from older versions.
  **Bump `CACHE_VERSION` on every future release.**

## Notes
- The service worker registers **only from production builds** (guarded by
  `import.meta.env.DEV`), so `npm run dev` / Vite HMR is unaffected.
- The admin / Filament panel uses a separate layout and is explicitly excluded
  by the service worker, so it is unaffected.

## Server deployment (Hostinger manual upload)
1. Extract `updates/v5.7.0.zip` at the **project root** — overlays the changed
   `resources/` files, the new `public/` static PWA files, and `.htaccess`
   (no migrations, no seeders, no `composer install`).
2. **Upload the Vite build AND the new PWA static files to BOTH public folders**
   (`public_html` **and** `/yemdatlive/public`) — copy `public/build/`,
   `public/manifest.json`, `public/service-worker.js`, `public/offline.html`,
   and `public/icons/` into both. If they land in only one, the site is unstyled
   and/or the PWA fails to register. The service worker **must** sit at the web
   root of the live document root so its scope covers all routes.
3. `php artisan optimize:clear` (refreshes the compiled `app-layout` view so the
   new PWA `<head>` tags render). Re-`optimize` afterwards if you cache in prod.
4. Returning visitors pick up the new service worker automatically on their next
   load (the `CACHE_VERSION` bump purges the old caches). Verify in the browser:
   DevTools ▸ Application ▸ Manifest (installable) and Service Workers (activated),
   then toggle Network ▸ Offline and reload a visited page.
