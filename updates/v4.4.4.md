# v4.4.4 — Fix "Forbidden" on Arabic Email Template Save (2026-04-23)

## What changed

### Fix: Hostinger WAF blocking Arabic content on email template save
Hostinger's WAF was rejecting any POST request containing Arabic Unicode characters in the raw
request body, returning a "Forbidden" page before Laravel could process it.

**Fix:** Arabic fields (`subject_ar`, `body_ar`) are now base64-encoded by JavaScript before the
form submits. The controller decodes them server-side before validation. Base64 is pure ASCII
and passes through the WAF without triggering any rules.

Affects all 8 email templates (WelcomeEmail, SignupOtpEmail, PasswordResetOtpEmail,
EventConfirmationEmail, EventReminderEmail, ContactUsAutoReplyEmail, TrainerAutoReplyEmail,
AccountDeletionOtpEmail).
