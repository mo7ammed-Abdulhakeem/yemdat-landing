# v4.8.0 тАФ Certificate System (2026-06-01)

## What changed

Phase 4: per-(member, event) **Certificates of Completion** as bilingual EN/AR PDFs with a
unique serial + QR code, an admin-editable design, member self-download, and a public
verification page.

- **Issuing** тАФ on each Event's Filament page, an **Attendees** panel lists registrants with
  *Mark attended* / *Mark completed* actions, a per-attendee **Issue certificate** (gated on
  "completed"), a bulk **Issue to completed**, and download. Issuance is one tested code path:
  `App\Actions\Certificates\IssueCertificate` (one cert per member+event, idempotent).
- **Certificates resource** (`/admin/certificates`) тАФ browse / view / **download** / **revoke**
  / reinstate every issued certificate.
- **Editable design** тАФ `/admin/manage-certificate-template` (gated `settings`): the certificate
  is an **HTML template in the DB** (like email templates) with `{tokens}`
  (`{member_name} {event_title_en} {event_title_ar} {date} {serial} {verify_url} {qr}
  {background_url} {issued_year}`), an optional background-image upload, paper size, and a
  **Download preview**. A polished bilingual default ships seeded.
- **PDF** тАФ rendered by **mPDF** (pure PHP, native Arabic/RTL тАФ safe on shared hosting). QR via
  mPDF's built-in `<barcode type="QR">` (`mpdf/qrcode`).
- **Member download** тАФ a "Download Certificate" button on each event card in `/my-profile`
  (owner-only route, blocks revoked).
- **Public verify** тАФ `/verify/{serial}` (the QR target): valid / revoked / not-found, bilingual.

Anchor schema: `event_member` gained `attended_at` + `completed_at`; new `certificates` and
`certificate_templates` tables. **94 tests green** (15 new).

> `/admincpanel` unchanged (still the fallback). Emailing certificates is intentionally out of
> scope for now (тЙИ100/day cap + Arabic-in-POST WAF).

## Server deployment
1. Upload files + new `vendor/` packages.
2. `php artisan migrate --force`
3. `php artisan db:seed --class=CertificateTemplateSeeder --force` (idempotent; safe тАФ won't clobber an edited template)
4. Ensure `storage/app/mpdf` is writable (mPDF temp; auto-created on first render).
5. `php artisan optimize:clear`
