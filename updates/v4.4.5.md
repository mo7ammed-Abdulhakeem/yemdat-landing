# v4.4.5 — Broadcast Top-up & Re-subscribe (2026-04-24)

## What changed

### Feature 1 — Send to New Registrants (Broadcast Top-up)
After sending a broadcast to event registrants, new members who register afterwards were
missing it. A "Send to New Registrants" button now appears on the broadcast detail page
for any event broadcast that has already been sent. It only emails members not already in
the recipient list, so existing recipients are never re-emailed.

**How it works:**
- Snapshot the current max recipient ID before inserting new rows
- Bulk-insert new recipient rows for members not in the existing list
- Dispatch `ProcessBroadcastJob` with the threshold ID — job filters `WHERE id > threshold`
- Increments `total_recipients` on the broadcast

### Feature 2 — Re-subscribe
Members who unsubscribed from broadcast emails can now re-subscribe via two paths:

**a) Unsubscribe confirmation page (email link)**
After unsubscribing (or on the "already unsubscribed" page), a "Re-subscribe to broadcast
emails" button appears. Clicking it sets `unsubscribed_at = null` on the member record.
Uses the existing broadcast tracking token — no login required.

**b) Member profile page**
The profile page (`/my-profile`) now shows an "Email Preferences" section. When unsubscribed
it shows a "Re-subscribe" button; when subscribed it shows the current status.
