# v5.7.1 — Custom "Install app" banner (2026-06-20)

Adds a branded, dismissible **install prompt** on top of the v5.7.0 PWA. Browsers
don't pop up an install dialog on their own, so the site now captures the
`beforeinstallprompt` event and shows its own bottom slide-up banner; tapping
**Install** opens the real browser install dialog. iPhone/Safari (which never
fires that event) gets a manual **"Share → Add to Home Screen"** hint instead.
Frontend-only — no migrations, no seeders, no new dependencies.

## What's new
- **Bottom install banner** (Chrome / Edge / Android): logo + short message +
  **Install** button + dismiss (✕). Slides up after the browser signals the site
  is installable (gated on engagement, so it won't flash on first paint).
- **iOS hint:** on iPhone/iPad Safari, a bilingual tip to add via the Share sheet.
- **Bilingual EN/AR + RTL** — copy and layout follow the page's `lang`/`dir`.
- **Non-naggy:** dismissal is remembered for **14 days**; the banner never shows
  when the app is already installed (running standalone) and hides on
  `appinstalled`.

## Files
- **New:** `resources/js/pwa-install.js` (the banner: event capture, UI, i18n,
  dismissal memory; inline-styled and self-contained).
- **Edited:** `resources/js/app.js` (lazy-loads the banner, production-only);
  `public/service-worker.js` (`CACHE_VERSION` → `yemdat-v5.7.1`).
- **Rebuilt:** `public/build/**` — new `app-Bd4sgkEI.js` plus a lazy chunk
  `pwa-install-DH9QdwHD.js`.

## Server deployment (Hostinger manual upload)

2. **Upload the updated `public/service-worker.js`** to the **top level** of both
