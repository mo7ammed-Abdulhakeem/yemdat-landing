# v4.7.1 — Filament Admin: More Resources (2026-05-31)

## What changed

Adds five more resources to the new Filament panel (Phase 3, continuing v4.7.0).
`/admincpanel` still remains the source of truth until parity.

- **Membership Tiers, Trainer Requests, Contacts, Email Templates, Users** resources
  (list / create / edit), generated from the models.
- **User resource form fixed** (the generated version was unsafe/broken):
  - `role` → Select (Admin / Super Admin).
  - `permissions` → CheckboxList with the real keys (`members, messages, events, posts,
    trainers, broadcasts, analytics, settings`) — the generated Textarea would have
    corrupted the JSON-array column.
  - `password` → required only on create; blank on edit keeps the current password;
    revealable. (The `User` model's `hashed` cast does the hashing.)

8 resources now live at `/admin`. `FilamentAdminTest` covers list pages for all of them
plus create-form rendering. Full suite green.

## Files changed
- `app/Filament/Resources/{MembershipTiers,TrainerRequests,Contacts,EmailTemplates,Users}/**` (NEW)
- `app/Filament/Resources/Users/Schemas/UserForm.php` (hand-fixed)
- `tests/Feature/FilamentAdminTest.php`

## Server deployment
Same as v4.7.0 (upload `app/`, `composer install`, `filament:assets:publish`, `optimize:clear`). No migrations.

## Verify
- `/admin` → all 8 resources in the sidebar; lists load; create/edit forms work.
- Create a user: role select + permission checkboxes + password (hashed) all behave correctly.
