# v4.7.2 — Filament Admin: Broadcasts + Shared Send Action (2026-05-31)

## What changed

Ports the **Broadcasts** subsystem to Filament — the most complex admin area — and
refactors the send logic into a shared action.

- **`App\Actions\Broadcasts\SendBroadcast`** (NEW): the recipient-collection + chunked-job
  dispatch logic, extracted so it's shared by both the legacy `BroadcastController` and the
  new Filament panel (mirrors `RegisterMemberForEvent`). `send()` and `sendToNew()`.
- **`BroadcastController`** refactored to call the action (same behavior/messages; the 11
  existing broadcast tests still pass).
- **Filament `EmailBroadcast` resource**:
  - Compose form: language + audience selects (reactive), event picker (shown for event
    audiences), bilingual subject + **RichEditor** body (the relevant language only), from
    name/email. New broadcasts are saved as `draft` with `sent_by = current admin`.
  - Table: subject, audience/language/status badges, recipient count, sent date.
  - Row actions: **Send** (drafts) and **Send to new** (sent event broadcasts) — both call
    `SendBroadcast` and notify; edit limited to drafts; delete also clears recipients.
  - **Permission-gated**: the whole resource requires the `broadcasts` permission
    (super_admins pass). Verified by test (non-permitted admin → 403).

Full suite green (76 tests).
