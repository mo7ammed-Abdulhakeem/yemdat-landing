# v4.9.0 — Member dashboard upgrade (Phase 5a) (2026-06-01)

## What changed

Phase 5 (UX/CX polish) begins with an upgrade to the member's `/my-profile` page: it now
opens with an at-a-glance **dashboard** instead of jumping straight into static details.

- **Stats row** — three cards: registered events (with an "X upcoming" sub-count),
  certificates earned, and a **profile-completeness** percentage with a progress bar.
- **Complete-your-profile nudge** — when the profile is < 100% complete, a card lists the
  exact missing fields as chips (e.g. Education level, Bio, LinkedIn) with a *Complete
  profile* button. Completeness is computed by `Member::profileCompletion()`.
- **Next-event highlight** — a banner surfaces the member's soonest upcoming event with its
  date, location, a live countdown ("N Days Left"), and the join link.
- **Upcoming / Past split** — the registered-events list is now grouped into **Upcoming** and
  **Past** sections (soonest upcoming first), each with its own count. The per-event card was
  extracted into a reusable `<x-profile.event-card>` component (keeps the certificate
  download button; hides the join link on past events).

Fully bilingual (EN/AR) and RTL-aware; before/after verified in both directions.

## Implementation notes
- `ProfileController::show()` loads registered events once and `partition()`s them into
  upcoming/past, passing `$upcomingEvents`, `$pastEvents`, `$nextEvent`, `$completion` to the
  view (no extra inline queries in the Blade).
- `Member::profileCompletion()` returns `percent` / `completed` / `total` / `missing` over the
  9 profile fields.
