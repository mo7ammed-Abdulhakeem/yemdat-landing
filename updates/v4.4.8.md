# v4.4.8 — Edit & Delete Broadcasts + Permission Control (2026-04-25)

## What changed

### Feature: Edit and delete draft broadcasts
Admins can now fix mistakes in a broadcast before sending it, and delete any broadcast permanently.

- **Edit** — only available on draft broadcasts (sent/sending broadcasts cannot be changed)
- **Delete** — available at any status; permanently removes the broadcast and all recipient records
- Edit button appears on the broadcast detail page and in the index list (draft rows only)
- Delete button appears on both the detail page and index list (all statuses)

### Security: Broadcasts now respect admin permissions
Any admin could previously access, compose, and send broadcasts regardless of their assigned permissions.

- All broadcast controller methods now return 403 if the user lacks the `broadcasts` permission
- The Broadcasts sidebar link is hidden for users without the permission
- "Broadcast Emails" checkbox added to Create User and Edit User permission forms
- Super admins are unaffected — they always have full access
