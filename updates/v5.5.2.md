# v5.5.2 — Redirect to the list after saving an edit (2026-06-10)

Small admin UX tweak. Saving an edit on **Events, Posts, and Members** now returns
you to that resource's **list** instead of keeping you on the edit screen.

## Fix
Added `getRedirectUrl()` → `getResource()::getUrl('index')` to the three edit pages.
(Members keeps its previous/next record navigation actions.)

## Files
- Edited: `app/Filament/Resources/Events/Pages/EditEvent.php`,
  `app/Filament/Resources/Posts/Pages/EditPost.php`,
  `app/Filament/Resources/Members/Pages/EditMember.php`.

## Server deployment (Hostinger manual upload)
1. Extract `updates/v5.5.2.zip` at the **project root** (overlays the 3 page files).
2. `php artisan optimize:clear` (or `view:clear` + `config:clear`).
3. No `npm run build`, no asset re-upload, no migration.
