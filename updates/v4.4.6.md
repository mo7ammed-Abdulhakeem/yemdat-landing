# v4.4.6 — Broadcast Daily-Chunk Rate Limiting (2026-04-24)

## What changed

### Fix: Hostinger 100 emails/day limit on broadcasts
Broadcasts were trying to send all emails at once, failing after ~100 recipients.
Now the job sends 90 emails per day and automatically reschedules itself for the
next day until all recipients are done. When the last batch finishes, the broadcast
flips from "Sending" to "Sent" automatically.

The daily limit is configurable via `BROADCAST_DAILY_LIMIT` in `.env` (default: 90).
When you upgrade to the Hostinger 1000/day plan, just set `BROADCAST_DAILY_LIMIT=900`.

### UI changes on broadcast detail page
- Added "Sending in daily batches" progress banner while sending
- Renamed "Total Sent" stat card to "Queued"
- Added new "Sent So Far" stat card showing actual emails delivered

## Files changed
- `app/Jobs/ProcessBroadcastJob.php`
- `app/Http/Controllers/Admin/BroadcastController.php`
- `app/Models/EmailBroadcastRecipient.php`
- `resources/views/admin/broadcasts/show.blade.php`
- `database/migrations/2026_04_24_230734_add_sent_at_to_email_broadcast_recipients_table.php`

## Server deployment
1. Upload all 5 files above (migration goes in `database/migrations/`)
2. `php artisan migrate`
3. `php artisan config:clear`

## Optional: set daily limit in .env
```
BROADCAST_DAILY_LIMIT=90
```
Leave unset to use the default of 90. Change to a higher number after upgrading
your Hostinger email plan.
