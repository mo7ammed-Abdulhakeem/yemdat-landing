# v4.10.1 — Be-a-Trainer form on the design system (2026-06-02)

Migrated the public **Be A Trainer** form (`/be-a-trainer`) to the shared design-system
components and fixed a layout bug.

- **Layout bug fixed:** a duplicate `</div>` split the form so only the first four fields sat
  inside the white card while Program Type, durations, agenda, agreement, and the submit button
  spilled onto the page background. The whole form is now one coherent `<x-ui.card>`.
- Fields use `<x-ui.input>` / `<x-ui.select>` / `<x-ui.label>` (semantic tokens, automatic
  `old()` repopulation + inline per-field validation errors).
- Validation errors now show as a single `<x-ui.alert variant="danger">` summary; the old
  `<x-alert-modal>` success popup was dropped in favour of the global toast (added in v4.9.1).
- Submit uses `<x-ui.button type="submit">`, so it inherits the loading-spinner / no-double-submit
  behaviour. Agreement checkbox restyled with tokens (`accent-primary`); page on `bg-surface`.
- The Quill rich-text agenda editor and all settings-driven labels/notes are preserved; RTL
  verified.

## Files changed (deploy to production)
- `resources/views/trainer.blade.php`
- **Rebuild assets**: `npm run build` + upload `public/build/*` (new utilities used).

Before/after verified EN + AR. 2 trainer tests green (field names unchanged). No migrations.
