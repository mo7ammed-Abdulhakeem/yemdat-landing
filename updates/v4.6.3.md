# v4.6.3 — Design System: Member Profile Pages (2026-05-31)

## What changed

- **Profile (`profile/show.blade.php`)**: the "Edit Profile", "Browse Events", "Contact Us",
  and "Re-subscribe" actions → `<x-ui.button>`; the membership pill → `<x-ui.badge>`. The
  existing card sections were left intact (already on-brand white cards).
- **Edit Profile (`profile/edit.blade.php`)**: the error summary → `<x-ui.alert>`; the
  Save/Cancel and "Request Account Deletion" actions → `<x-ui.button>` (danger variant).

> Follow-up (queued, not in this release): convert the Edit Profile form **fields**
> (inputs/selects/TomSelect) to `<x-ui.input>/<x-ui.select>` for full parity with the
> register form, and migrate the four OTP/verify forms (account-deletion confirm,
> verify-email, verify-password-otp, set-password).

## Files changed (deploy to production)
- `resources/views/profile/show.blade.php`
- `resources/views/profile/edit.blade.php`

## Server deployment
1. Upload the two files. 2. `npm run build` + upload `public/build/`. 3. `php artisan optimize:clear`. No migrations.

## Verify
- Log in → `/my-profile` and `/my-profile/edit` render with consistent buttons/badges; edit/save/delete actions work.
