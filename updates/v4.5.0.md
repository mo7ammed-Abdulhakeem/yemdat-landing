# v4.5.0 — Testing Foundation, CI & Bug Fixes (2026-05-31)

## What changed

This release starts the platform's growth roadmap with a developer testing foundation
and three production bug fixes. Most of it is internal tooling (tests, CI, code style)
that is **not** deployed to the server — only the four bug-fix files listed below go to
production.

### Bug fixes (production)
1. **Event confirmation email showed a blank location.** `EventController` read
   non-existent `location_en`/`location_ar` columns (the events table has a single
   `location`). Now uses `$event->location`.
2. **Verified members were never marked verified.** `VerificationController` set
   `email_verified_at` via mass assignment, but the column isn't fillable, so it was
   silently dropped. Now set explicitly with `forceFill(...)->save()`.
3. **Profile update could return a 500.** Saving a profile without `specialty` wrote
   `null` to the NOT NULL `members.specialty` column. `specialty` is now required
   server-side (matching the edit form and registration), so it fails validation cleanly.

### Developer tooling (not deployed to the server)
- **69 feature tests** (PHPUnit on MySQL) covering registration/OTP, password reset,
  account deletion, claim-profile, events, contact, trainer requests, profile update,
  admin auth + permission gating, and the broadcast system (send, daily-chunk job,
  open tracking, unsubscribe/resubscribe).
- **Schema baseline** `database/schema/mysql-schema.sql` — makes fresh installs/CI work
  and fixes a latent migration-ordering bug (production schema is unchanged).
- **GitHub Actions CI** (`.github/workflows/ci.yml`): tests on MariaDB + Pint on new files.
- **Pint** config (`pint.json`); thin **Action layer** `RegisterMemberForEvent`.

2. No migrations needed. No artisan commands needed.
3. Recommended: `php artisan optimize:clear`.

2. Complete signup OTP → the member's `email_verified_at` is populated.
3. Edit your profile and save → no 500; clearing specialty shows a validation error.
