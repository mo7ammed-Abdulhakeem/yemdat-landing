# v5.7.2 — "University Major" relabel + logical grouping (2026-07-09)

Renames the member "Specialty" field to **University Major** everywhere it is
shown — signup, member profile, admin panel, CSV exports, and analytics — so its
meaning is unambiguous for members, admins, and analytics. The field is also
**regrouped with Education** (instead of Membership) across the signup wizard, the
member profile, and the admin view/edit screens, which reads more logically. Also
fixes a signup-wizard bug where a University Major validation error reopened the
wrong step. Display/label change only — the `specialty` DB column, model logic,
validation rules, and analytics grouping are unchanged.

## What's new
- **Consistent label "University Major" / "التخصص الجامعي"** on the signup form,
  member profile (view + edit), admin members form/table/filter/detail, the CSV
  export header, and the analytics "Members by university major" chart heading.
- **Logical grouping:** University Major now sits next to **Education Level**
  (signup Step 2, member profile, admin "Member" section / form) rather than with
  Membership.
- **Admin taxonomy screen** (Settings → the list of majors) is relabeled from
  "Specialties" to **University Majors** (nav item, list title, "New …" button,
  breadcrumbs) for consistency.
- **Bugfix (signup wizard):** a `specialty` / `specialty_other` validation error
  now reopens **Step 2** (where the field lives) instead of Step 3.

## Server deployment (Hostinger manual upload)
1. **Extract this bundle at the project root**, overwriting the edited `app/`,
   `resources/`, and `lang/` files.
2. **No migrations, no seeders, no `composer install`, no new dependencies.**
3. Run **`php artisan optimize:clear`** (clears compiled Blade views, config, and
   route/translation caches so the new labels render).
4. **Vite build:** unchanged from v5.7.1 — you do **not** need to re-upload
   `public/build`. (The bundle still includes it under `public/build/` if you want
   a full sync; if you do upload, it must go to **BOTH** `public_html/build/` and
   `…/public/build/`.)
5. Verify: signup form Step 2 shows "University Major" next to Education; submitting
   an empty major reopens Step 2 (not Step 3); admin Members list/detail/export and
   the analytics chart all read "University Major"; toggle AR/EN to confirm both.
