# v4.7.0 — Filament Admin Panel: Foundation (2026-05-31)

## What changed

Starts **Phase 3**: a new Filament v5 admin panel, built **alongside** the existing
hand-built `/admincpanel` (which stays live until the new panel reaches parity).

- Installed `filament/filament` v5 (+ Livewire).
- New panel at **`/admin`** scoped to the existing `web` guard + `User` model.
  - Branded "Yemdat Admin", brand-brown primary color.
  - `SetLocale` middleware added so the panel is bilingual / RTL like the rest of the app.
  - Access gated: `User` now implements `FilamentUser::canAccessPanel()` → only `admin` /
    `super_admin` roles. (The `users` table is staff-only.)
- First resources generated (list/create/edit/table/form): **Members, Events, Posts**.
- Tests: `FilamentAdminTest` (guest redirected; admin loads dashboard + members/events/posts).
  Full suite green (73 tests).

> The existing `/admincpanel` is untouched and remains the source of truth until all
> resources (Broadcasts + send/track actions, Settings, Users/permissions, Trainer
> Requests, Contacts, Email Templates) and the analytics widgets are ported.

## Server deployment (heavier than usual — Filament needs Composer on the server)
1. Upload the changed `app/`, `bootstrap/providers.php`, `composer.json`, `composer.lock`.
2. On the server: `composer install --no-dev --optimize-autoloader` (installs `vendor/filament`).
3. `php artisan filament:assets:publish` (generates `public/css|js|fonts/filament`).
4. `php artisan filament:optimize` (or at least `php artisan optimize:clear`).
5. No migrations needed yet.
6. Create/confirm an admin: ensure a `users` row exists with role `admin`/`super_admin`.
