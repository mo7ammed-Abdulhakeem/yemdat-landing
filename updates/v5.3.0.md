# v5.3.0 — Trainer-role audit fixes + page visibility manager (2026-06-05)

Two things: tighten the rough edges a QA audit found in the new Trainer role, and add
an admin control to switch public pages on/off (e.g. keep **Learning Paths** hidden
until it's ready to go live).

## Part A — Trainer role fixes (from the QA audit)
- **Permissions field no longer misleading for trainers.** In the Users form the
  granular permissions only gate `/admin` (which trainers can't open), so they did
  nothing for a `trainer`-role user. The CheckboxList is now **hidden when role =
  Trainer**, replaced by a short note explaining trainers use the separate `/trainer`
  dashboard. ([UserForm.php](app/Filament/Resources/Users/Schemas/UserForm.php))
- **Resend trainer invite.** A new **"Resend trainer invite"** action (member view +
  members table, shown for trainers) re-sends a fresh signed set-password link —
  recovering from a failed or expired invite without revoke + re-promote. Invite
  logic extracted to a reusable `App\Actions\Trainers\SendTrainerInvite`
  (used by both promote and resend).
- **Bilingual trainer-panel labels** — "My Events" / "Certificates" now localize
  (EN/AR) via `lang/{en,ar}/trainer.php` + `getNavigationLabel()`.
- **More tests** — resend sends an invite; **Revoke must not delete an admin-role
  linked user**; "My Events" is read-only (no create/edit route).

## Part B — Page visibility manager
Admins can turn public pages on/off from **Settings → Page Visibility**. When a page
is off it's **hidden from the menu** and returns **404** if opened directly.

- **Registry** `config/pages.php` — manageable pages with their route names + a default
  state. Ships with: **Learning Paths (default OFF — hidden until made live)**, News,
  Membership, Be-a-Trainer (default on). Core pages (home/events/contact) are
  deliberately not managed so links never break.
- **State** lives in the existing `settings` table (`page_{key}_active` = '1'/'0'),
  read via `App\Support\PageVisibility` (config default until an admin saves).
- **Enforcement** — `App\Http\Middleware\EnsurePageActive` (appended to the `web`
  group) 404s a disabled managed page; no-op for everything else. Nav links are wrapped
  in a new `@pageactive('key') … @endpageactive` Blade conditional.
- **Admin UI** — `App\Filament\Pages\ManagePages` (Settings group, gated `settings`),
  a Toggle per registry entry; mirrors the existing `ManageSettings` page pattern.

## Server deployment (Hostinger manual upload)
1. Upload changed PHP + Blade + `config/pages.php` + lang files.
2. **No migration** — page flags live in the existing `settings` table (created on
   first save; config-default until then).
3. `php artisan optimize:clear` (re-`optimize` if cached — class-based middleware +
   config registry, so `route:cache`/`config:cache` stay safe). If you cache views,
   `php artisan view:clear` (a new Blade directive was added).
4. No `npm run build`, no `filament:assets:publish`.

**Note:** Learning Paths now ships **hidden**. Flip it on in **Settings → Page
Visibility** when you're ready to launch it. No new dependencies.
