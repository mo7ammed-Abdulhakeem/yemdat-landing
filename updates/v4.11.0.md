# v4.11.0 — Unified specialties + analyst dashboard (2026-06-02)

Two linked deliverables: clean the fragmented "major" data and visualize it.

## Part B — Unified specialty taxonomy
The old signup offered 4 options + free-text "Other", so **63% of members (420/663) were "Other"**
and the field fragmented into 207 variants across EN/AR/misspellings.

- **`specialties` table + admin resource** — new bilingual, admin-managed taxonomy (15 categories:
  Data Analytics/Science/Engineering/Management/Governance, Business Intelligence, AI & ML, IT,
  Computer Science, Software Engineering, Cybersecurity, Statistics & Math, Business & Management,
  Accounting & Finance, Other). Managed at **Settings → Specialties** (gated `settings`), with a
  live **member count** per specialty. Seeded idempotently by `SpecialtySeeder`.
- **Single source of truth** — signup (`auth/register`), profile edit, and the Filament Member
  form/table/infolist all read the list; `members.specialty` now stores the **slug**.
  `specialty` is validated against active slugs (`Rule::exists`), `Other` requires a detail.
  `Member::specialtyLabel` renders the localized name (prefers the free-text detail for "Other").
- **Data cleanup** — `App\Support\SpecialtyNormalizer` maps legacy/variant/free-text values
  (EN + AR, acronyms) to canonical slugs; a backfill migration applied it to all members,
  preserving `specialty_other`. **Result: "Other" 420 → 207**, and real fields surfaced
  (IT 70, Computer Science 42, Business 31, Software Eng 17, Cybersecurity 16, Accounting 15…).

## Part A — Analyst dashboard
- **Home dashboard** kept light (custom `App\Filament\Pages\Dashboard`): just the KPI overview
  (members + 30-day growth %, new 7d, events, **event registrations**, **certificates issued**,
  posts, trainer requests, messages).
- **Dedicated Analytics page** (`/admin/analytics`, gated `analytics`) with 6 Chart.js widgets:
  member growth (12-mo line), members by specialty, gender, education, top countries, top events
  by registrations. Bilingual: chart category labels localize (EN/AR/RTL).

## Server deployment
1. Upload files + `npm run build` is **not** required (no public CSS/JS changes; forms reuse
   existing components). Charts use Filament assets — run `php artisan filament:assets:publish` to be safe.
2. `php artisan migrate --force` (creates `specialties`, runs the member backfill).
3. `php artisan db:seed --class=SpecialtySeeder --force` (idempotent — seeds the 15 categories).
4. `php artisan optimize:clear` (then re-`optimize` if you cache in prod).

No new dependencies. Back up the DB before the backfill (it rewrites `members.specialty`; original
free-text remains in `specialty_other`).
