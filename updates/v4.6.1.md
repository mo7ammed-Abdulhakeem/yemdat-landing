# v4.6.1 — Design System: Register Form + Home Hero (2026-05-31)

## What changed

Continues the design-system rollout (see `v4.6.0`) by migrating the two remaining
public pages.

### Register form (`auth/register.blade.php`)
- Migrated all fields to the component library: text inputs → `<x-ui.input>`,
  dropdowns → `<x-ui.select>`, submit → `<x-ui.button variant="accent">`, card wrapper
  → `<x-ui.card>`, error summary → `<x-ui.alert>`.
- Inputs now use the consistent white/rounded field style (previously gray fills),
  with inline per-field validation errors.
- **Behavior preserved**: TomSelect searchable country dropdown, the phone code+number
  combo, and the Alpine "Other specialty" toggle all still work. TomSelect styling was
  aligned to the design tokens.

### Home hero (`welcome.blade.php`)
- Rebuilt the hero on the design system and **removed ~90 lines of bespoke inline `<style>`**.
- Title now uses the brand font (Instrument Sans, extrabold) instead of Arial Black;
  description uses the warm `ink-soft` token; the secondary CTA is the standard `outline`
  button. Layout (label · title · description · two CTAs) unchanged.

## Files changed (deploy to production)
- `resources/views/auth/register.blade.php`
- `resources/views/welcome.blade.php`

## Server deployment
1. Upload the two files above.
2. **Rebuild assets** (view classes changed): `npm run build`, then upload `public/build/`.
3. `php artisan optimize:clear`.
4. No migrations needed.

## Verify
1. `/register` — fields styled consistently; country search, phone, and "Other specialty"
   toggle all work; submitting with errors shows inline messages.
2. `/` (home) — hero renders with the brand title/description and both CTA buttons, LTR + RTL.
