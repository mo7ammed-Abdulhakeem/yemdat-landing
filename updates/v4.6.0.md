# v4.6.0 — Unified Design System (foundation) + Member-Login Fix (2026-05-31)

## What changed

Introduces a reusable, RTL-aware **design system** and starts migrating the public
site onto it. Also fixes a member-login redirect bug. See `DESIGN.md`.

### Design tokens & Arabic font
- Semantic design tokens in `resources/css/app.css` (`primary/accent/surface/ink/border`
  + status colors, radii, shadows) layered on the existing brand palette.
- Arabic web font **Tajawal** loaded and auto-applied (`font-arabic`) on RTL pages in
  both the public and admin layouts — improves Arabic readability.

### Reusable components (`resources/views/components/`)
- `ui/`: `button, card, badge, alert, label, input, textarea, select, page-header`.
- `events/event-card` (variants: live | upcoming | past) and `posts/post-card`.

### Pages migrated to the design system
- **Contact** form, **Events** index (3 duplicated card blocks → one `event-card`,
  269→~70 lines), **News** index grid (→ `post-card`), **Login**, **Forgot Password**.
- Inputs now show inline per-field validation errors.

### Bug fix: member-login redirect
- Unauthenticated visitors to member-only pages were sent to the **admin** login.
  `bootstrap/app.php` now redirects member-area guests to the member login
  (`/login`) and admin-area guests to the admin login (`/admincpanel/login`).

> Not yet migrated (planned for the next release): the **register** form (TomSelect /
> multi-section) and the **home hero** (bespoke section) — each needs its own review.

## Server deployment
1. Upload the production files above (create the new `components/{ui,events,posts}` dirs).
2. **Rebuild assets** (the CSS source changed): `npm install` (if needed) then `npm run build`,
   and upload the regenerated `public/build/` directory.
3. `php artisan optimize:clear`.
4. No migrations needed.
