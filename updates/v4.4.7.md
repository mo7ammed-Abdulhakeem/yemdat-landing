# v4.4.7 — Broadcast Permission Control (2026-04-25)

## What changed

### Security: Broadcasts now respect admin permissions
The broadcast system previously had no authorization checks — any admin could access,
compose, and send broadcasts regardless of their assigned permissions.

- All 5 broadcast controller methods now abort(403) if the user lacks the `broadcasts` permission
- The Broadcasts sidebar link is now hidden for users without the permission
- "Broadcast Emails" checkbox added to Create User and Edit User permission forms

Super admins are unaffected — `hasPermission()` already returns true for super_admin unconditionally.

## Files changed
- `app/Http/Controllers/Admin/BroadcastController.php`
- `resources/views/components/admin/sidebar-content.blade.php`
- `resources/views/admin/users/create.blade.php`
- `resources/views/admin/users/edit.blade.php`

## Server deployment
1. Upload all 4 files above
2. No migrations or artisan commands needed
3. After upload: go to Manage Users → edit any admin → "Broadcast Emails" checkbox now appears

## Important note
Existing admin users will NOT automatically have the broadcasts permission after this update.
You must manually grant it in Manage Users → Edit → check "Broadcast Emails" for each admin
who should have access. Super admins keep full access automatically.
