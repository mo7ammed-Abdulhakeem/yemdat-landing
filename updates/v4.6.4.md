# v4.6.4 — Design System: Profile Edit Fields + OTP/Verify Forms (2026-05-31)

## What changed

Completes the public/member design-system migration (the follow-ups noted in v4.6.3).

- **Edit Profile (`profile/edit.blade.php`)**: all form fields migrated to
  `<x-ui.input>/<x-ui.select>/<x-ui.textarea>` (white rounded fields). The locked email,
  phone code+number combo, TomSelect country, the Alpine-free specialty toggle, and the
  LinkedIn icon-prefixed input are preserved; TomSelect styling aligned to the design tokens.
- **OTP / verification forms** migrated to `<x-ui.card>/<x-ui.alert>/<x-ui.button>` (the
  large centered OTP inputs are kept, now using the token focus styles):
  - `auth/verify-email.blade.php` (signup OTP)
  - `auth/verify-password-otp.blade.php` (password-reset OTP)
  - `auth/set-password.blade.php` (claim / reset set-password)
  - `profile/delete-confirm.blade.php` (account-deletion OTP)

All four flows were exercised end-to-end (login, forgot-password, claim-profile,
delete-request) to confirm they render correctly; `php artisan test` stays green (69 tests).
